## 1. Overview

MemoizeLib is an annotation-based memoization library for Android that allows developers to cache method results by simply adding `@Memoize` to a method. A Gradle plugin performs ASM bytecode transformation at build time to inject all cache mechanics transparently -- no call-site changes, no wrapper classes, no boilerplate.

**Developer experience:**

```java
// Before: no memoization
public boolean search(int key) { /* expensive traversal */ }
public void insert(int data)   { /* mutates state */ }

// After: add two annotations -- that's it
@Memoize(maxSize = 64)
public boolean search(int key) { /* expensive traversal */ }

@CacheInvalidate
public void insert(int data)   { /* mutates state */ }
```

No changes to callers. No wrapper classes. The build-time transformation handles everything.

**Key design properties:**
- **Transparent**: No call-site changes required. The method signature is unchanged.
- **Java + Kotlin**: Works on both Java and Kotlin Android code (operates at bytecode level).
- **Lightweight**: All cache logic lives in a small runtime library (~7 classes). The ASM injection is a thin shell.
- **Correctness-first**: `@CacheInvalidate` clears all caches on the instance when mutating methods are called.
- **Configurable**: Cache size, eviction policy, TTL, thread safety, and stats are all annotation parameters.

## 2. Architecture

The library uses a two-layer architecture:

```
 Source Code          Compile Time              Build Time (AGP)           Runtime
 ==========          ============              ================           =======

 @Memoize     --->   KSP validates      --->   ASM transforms     --->   MemoDispatcher
 @CacheInvalidate    (errors/warnings)          bytecode                  checks cache,
                                                (adds fields,             computes on miss,
                                                 injects cache logic)     stores result
```

**Layer 1 -- Compile-time validation (KSP):**
The KSP processor scans for `@Memoize` annotations and emits compile-time errors (void return, abstract method) and warnings (mutable parameter types). It generates no code.

**Layer 2 -- Build-time bytecode transformation (AGP + ASM):**
The Gradle plugin hooks into the Android build pipeline via the AGP Instrumentation API. An ASM `ClassVisitor` reads each class, detects annotated methods, and injects:
- `MemoCacheManager` and `MemoDispatcher` fields
- Field initialization in constructors
- Cache-check bytecode at method entry
- Cache-store bytecode before method returns
- `cacheManager.invalidateAll()` or `cacheManager.invalidate(String[])` calls in `@CacheInvalidate` methods (full or selective invalidation)

All actual cache logic (LRU eviction, key construction, TTL, thread safety) is in the **runtime library**, not in generated bytecode. The bytecode is a thin coordination layer that calls into the runtime.

## 4. Annotations API

### `@Memoize`

**File:** [`memoize-annotations/src/main/java/dev/memoize/annotations/Memoize.java`](memoize-annotations/src/main/java/dev/memoize/annotations/Memoize.java)

Marks a method for automatic memoization. The method must have a non-void return type.

```java
@Retention(RetentionPolicy.CLASS)
@Target(ElementType.METHOD)
public @interface Memoize {
    int maxSize() default 128;              // Max cache entries
    long expireAfterWrite() default -1;     // TTL in ms (-1 = no expiry)
    CacheScope scope() default CacheScope.INSTANCE;
    EvictionPolicy eviction() default EvictionPolicy.LRU;
    ThreadSafety threadSafety() default ThreadSafety.CONCURRENT;
    boolean recordStats() default false;    // Track hit/miss rates
}
```

| Parameter | Default | Description |
|-----------|---------|-------------|
| `maxSize` | 128 | Maximum entries before LRU eviction kicks in |
| `expireAfterWrite` | -1 (disabled) | Time-to-live in milliseconds after a cache entry is written |
| `scope` | `INSTANCE` | `INSTANCE` = per-object cache, `CLASS` = static/shared cache |
| `eviction` | `LRU` | `LRU` = least recently used eviction, `NONE` = unbounded |
| `threadSafety` | `CONCURRENT` | `CONCURRENT` = ConcurrentHashMap, `SYNCHRONIZED` = synchronized blocks, `NONE` = no sync |
| `recordStats` | false | When true, tracks hit/miss/eviction counts via `CacheStats` |

**Retention is `CLASS`**: The annotation is written into `.class` bytecode (readable by ASM at build time) but not available via runtime reflection. This means it's stripped from the final APK, adding zero runtime overhead.

### `@CacheInvalidate`

**File:** [`memoize-annotations/src/main/java/dev/memoize/annotations/CacheInvalidate.java`](memoize-annotations/src/main/java/dev/memoize/annotations/CacheInvalidate.java)

Marks a mutating method as a cache invalidator. Supports both **full invalidation** (all caches) and **selective invalidation** (specific methods only).

```java
@Retention(RetentionPolicy.CLASS)
@Target(ElementType.METHOD)
public @interface CacheInvalidate {
    String[] value() default {};  // Empty = invalidate ALL caches
}
```

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `value` | `String[]` | `{}` (empty) | Names of `@Memoize` methods to invalidate. Empty = all. |

**Full invalidation** (no arguments -- clears all caches):
```java
@CacheInvalidate
public void resetAll() { /* clears everything */ }
```

**Selective invalidation** (only named caches are cleared):
```java
// Only invalidates search and length; other caches (e.g., describe) are preserved
@CacheInvalidate({"search", "length"})
public void insert(int data) { /* modifies the data structure */ }

// Only invalidates the describe cache
@CacheInvalidate("describe")
public void updateDescriptions() { /* modifies description data */ }
```

Selective invalidation is a performance optimization: when a class has many memoized methods and a mutation only affects specific cached queries, you avoid unnecessarily clearing unrelated caches.

At the bytecode level:
- No arguments → ASM generates `cacheManager.invalidateAll()`
- With arguments → ASM generates `cacheManager.invalidate(new String[]{"search", "length"})`

### `@CacheKey`

**File:** [`memoize-annotations/src/main/java/dev/memoize/annotations/CacheKey.java`](memoize-annotations/src/main/java/dev/memoize/annotations/CacheKey.java)

Specifies which field of a complex parameter to use for cache key construction.

```java
@Memoize
public Route findRoute(@CacheKey("id") Location start, @CacheKey("id") Location end) {
    // Only start.id and end.id are used as cache keys
}
```

> **Note:** `@CacheKey` field extraction is defined in the annotation API but not yet implemented in the ASM transformation. Currently, entire parameter objects are used as keys via `Arrays.deepHashCode`/`Arrays.deepEquals`. See [Limitations](#11-current-limitations).

### Enums

**Files:**
- [`CacheScope.java`](memoize-annotations/src/main/java/dev/memoize/annotations/CacheScope.java): `INSTANCE` (per-object) or `CLASS` (static/shared)
- [`EvictionPolicy.java`](memoize-annotations/src/main/java/dev/memoize/annotations/EvictionPolicy.java): `LRU` or `NONE`
- [`ThreadSafety.java`](memoize-annotations/src/main/java/dev/memoize/annotations/ThreadSafety.java): `NONE`, `SYNCHRONIZED`, or `CONCURRENT`

## 5. Runtime Library

The runtime library provides the cache infrastructure that the ASM-injected bytecode calls into. All classes are in the `dev.memoize.runtime` package.

### 5.1 `MemoCache<K, V>` (Interface)

**File:** [`memoize-runtime/.../MemoCache.java`](memoize-runtime/src/main/java/dev/memoize/runtime/MemoCache.java)

```java
public interface MemoCache<K, V> {
    V get(K key);
    void put(K key, V value);
    void clear();
    int size();
}
```

### 5.2 `LruMemoCache<K, V>`

**File:** [`memoize-runtime/.../LruMemoCache.java`](memoize-runtime/src/main/java/dev/memoize/runtime/LruMemoCache.java)

LRU cache backed by `LinkedHashMap` with access-order iteration. When size exceeds `maxSize`, the least recently accessed entry is evicted.

```java
// Internal: LinkedHashMap with access-order and removeEldestEntry override
new LinkedHashMap<K, V>(Math.min(maxSize, 16), 0.75f, true) {
    @Override
    protected boolean removeEldestEntry(Map.Entry<K, V> eldest) {
        boolean shouldRemove = size() > maxSize;
        if (shouldRemove && stats != null) stats.recordEviction();
        return shouldRemove;
    }
};
```

Thread safety is via `synchronized` methods. All `get`, `put`, `clear`, and `size` operations are synchronized.

### 5.3 `ConcurrentMemoCache<K, V>`

**File:** [`memoize-runtime/.../ConcurrentMemoCache.java`](memoize-runtime/src/main/java/dev/memoize/runtime/ConcurrentMemoCache.java)

Unbounded cache backed by `ConcurrentHashMap`. Lock-free reads, bucket-level write locks. No automatic eviction.

### 5.4 `CacheKeyWrapper`

**File:** [`memoize-runtime/.../CacheKeyWrapper.java`](memoize-runtime/src/main/java/dev/memoize/runtime/CacheKeyWrapper.java)

Wraps method arguments (`Object[]`) into a proper cache key with correct `hashCode()`/`equals()` semantics.

```java
public final class CacheKeyWrapper {
    public static final CacheKeyWrapper EMPTY = new CacheKeyWrapper(new Object[0]);

    private final Object[] args;
    private final int hashCode;

    public CacheKeyWrapper(Object[] args) {
        this.args = args;
        this.hashCode = Arrays.deepHashCode(args);  // handles nested arrays, nulls
    }

    @Override
    public boolean equals(Object obj) {
        if (this == obj) return true;
        if (!(obj instanceof CacheKeyWrapper)) return false;
        return Arrays.deepEquals(this.args, ((CacheKeyWrapper) obj).args);
    }
}
```

Key design decisions:
- Uses `Arrays.deepHashCode`/`Arrays.deepEquals` to handle primitives, objects, nested arrays, and nulls uniformly.
- Pre-computes hash code in constructor (immutable after creation).
- `EMPTY` singleton for zero-argument methods avoids allocation.

### 5.5 `MemoDispatcher`

**File:** [`memoize-runtime/.../MemoDispatcher.java`](memoize-runtime/src/main/java/dev/memoize/runtime/MemoDispatcher.java)

The core class that the ASM-injected bytecode calls. Each `@Memoize`-annotated method gets its own `MemoDispatcher` instance.

**Key methods called by ASM bytecode:**

```java
// 1. Build a cache key from boxed method arguments
public CacheKeyWrapper buildKey(Object[] args)

// 2. Check cache for a hit (returns cached value, NULL_SENTINEL, or null for miss)
public Object getIfCached(CacheKeyWrapper key)

// 3. Store a computed result
public Object putInCache(CacheKeyWrapper key, Object result)

// 4. Unwrap NULL_SENTINEL back to null
public static Object unwrap(Object cached)
```

**Null handling via sentinel pattern:**

A critical design issue: how to distinguish "the cache has no entry for this key" (miss) from "the cache has an entry whose value is null" (hit). The `MemoDispatcher` uses a sentinel object:

```java
private static final Object NULL_SENTINEL = new Object();

// In putInCache:
Object toStore = result != null ? result : NULL_SENTINEL;
cache.put(key, toStore);

// In getIfCached:
Object cached = cache.get(key);
if (cached != null) {
    stats.recordHit();
    return cached;   // caller must call unwrap() to convert sentinel back to null
}
stats.recordMiss();
return null;         // null means "cache miss"
```

**TTL support:**

When `expireAfterWrite > 0`, the dispatcher maintains a parallel timestamp cache. On `getIfCached()`, it checks if the entry has expired:

```java
if (timestamps != null) {
    Long writeTime = timestamps.get(key);
    if (writeTime != null) {
        long elapsed = System.nanoTime() - writeTime;
        if (elapsed > expireAfterWriteMs * 1_000_000L) {
            // Expired: evict and treat as miss
            cache.put(key, null);
            timestamps.put(key, null);
        }
    }
}
```

### 5.6 `MemoCacheManager`

**File:** [`memoize-runtime/.../MemoCacheManager.java`](memoize-runtime/src/main/java/dev/memoize/runtime/MemoCacheManager.java)

Manages all `MemoDispatcher` instances for a single object. The ASM transformation adds one `MemoCacheManager` field per instrumented class.

```java
public final class MemoCacheManager {
    private final ConcurrentHashMap<String, MemoDispatcher> dispatchers = new ConcurrentHashMap<>();

    public void register(String methodName, MemoDispatcher dispatcher) { ... }
    public MemoDispatcher getDispatcher(String methodName) { ... }

    // Called by @CacheInvalidate (no args) -- clears ALL caches
    public void invalidateAll() {
        for (MemoDispatcher dispatcher : dispatchers.values()) {
            dispatcher.invalidate();
        }
    }

    // Called by @CacheInvalidate({"search", "length"}) -- selective invalidation
    public void invalidate(String[] methodNames) {
        for (String name : methodNames) {
            MemoDispatcher dispatcher = dispatchers.get(name);
            if (dispatcher != null) {
                dispatcher.invalidate();
            }
        }
    }
}
```

The manager supports two modes:
- **Full invalidation** (`invalidateAll()`): clears every cache. Used when `@CacheInvalidate` has no arguments.
- **Selective invalidation** (`invalidate(String[])`): clears only the named caches. Used when `@CacheInvalidate({"method1", "method2"})` specifies targets. Unknown names are silently ignored.

### 5.7 `CacheStats`

**File:** [`memoize-runtime/.../CacheStats.java`](memoize-runtime/src/main/java/dev/memoize/runtime/CacheStats.java)

Thread-safe statistics tracker using `AtomicLong` counters:

```java
public final class CacheStats {
    private final AtomicLong hitCount = new AtomicLong();
    private final AtomicLong missCount = new AtomicLong();
    private final AtomicLong evictionCount = new AtomicLong();

    public double getHitRate() {
        long requests = hitCount.get() + missCount.get();
        return requests == 0 ? 0.0 : (double) hitCount.get() / requests;
    }
}
```

Stats are only tracked when `@Memoize(recordStats = true)` is set. When disabled (the default), the `stats` field is `null` and all stat-recording calls are skipped.

## 6. Build-Time Bytecode Transformation (ASM)

This is the core mechanism that makes `@Memoize` transparent. The transformation happens during the Android build via the AGP Instrumentation API.

### 6.1 Plugin Registration

**File:** [`memoize-gradle-plugin/.../MemoizePlugin.kt`](memoize-gradle-plugin/src/main/kotlin/dev/memoize/plugin/MemoizePlugin.kt)

```kotlin
class MemoizePlugin : Plugin<Project> {
    override fun apply(project: Project) {
        val androidComponents = project.extensions.findByType(AndroidComponentsExtension::class.java)
        androidComponents?.onVariants { variant ->
            variant.instrumentation.transformClassesWith(
                MemoizeClassVisitorFactory::class.java,
                InstrumentationScope.PROJECT   // Only transform project classes, not dependencies
            ) {}
            variant.instrumentation.setAsmFramesComputationMode(
                FramesComputationMode.COMPUTE_FRAMES_FOR_INSTRUMENTED_METHODS
            )
        }
    }
}
```

This registers the `MemoizeClassVisitorFactory` to be called for every `.class` file in the project during the `transformDebugClassesWithAsm` / `transformReleaseClassesWithAsm` Gradle task.

### 6.2 Class Filtering

**File:** [`memoize-gradle-plugin/.../MemoizeClassVisitorFactory.kt`](memoize-gradle-plugin/src/main/kotlin/dev/memoize/plugin/MemoizeClassVisitorFactory.kt)

```kotlin
override fun isInstrumentable(classData: ClassData): Boolean {
    val className = classData.className
    return !className.startsWith("android.") &&
           !className.startsWith("androidx.") &&
           !className.startsWith("kotlin.") &&
           !className.startsWith("java.") &&
           !className.startsWith("dev.memoize.runtime.") &&
           !className.startsWith("dev.memoize.annotations.")
}
```

All project classes are visited (since AGP's `ClassData` doesn't expose method-level annotations). The `MemoizeClassVisitor` checks for annotations inside and passes through unmodified classes that have none.

### 6.3 Two-Pass Transformation Strategy

**File:** [`memoize-gradle-plugin/.../MemoizeClassVisitor.kt`](memoize-gradle-plugin/src/main/kotlin/dev/memoize/plugin/MemoizeClassVisitor.kt)

The transformation uses the ASM **tree API** (`ClassNode`) for a two-pass approach:

```
Pass 1 (Tree API):
  Read class into ClassNode
  → Scan all methods for @Memoize / @CacheInvalidate annotations
  → Collect metadata (method names, descriptors, maxSize values)
  → Add MemoCacheManager + MemoDispatcher fields
  → Patch constructors (insert initialization before RETURN)
  → Write modified ClassNode to bytes

Pass 2 (Visitor API):
  Re-read modified bytes
  → For @Memoize methods: inject cache-check at entry, cache-store at exit
  → For @CacheInvalidate methods: inject invalidateAll() at exit
  → Write to next ClassVisitor (output)
```

**Why two passes?** The single-pass visitor pattern processes methods in file order. Constructors (`<init>`) appear before other methods, but we need to know which methods are memoized to generate the correct constructor initialization. The tree API lets us scan all methods first, then patch the constructor with full knowledge.

### 6.4 Phase 1: Annotation Scanning

```kotlin
for (method in classNode.methods) {
    val allAnnotations = (method.visibleAnnotations.orEmpty()) + (method.invisibleAnnotations.orEmpty())
    for (ann in allAnnotations) {
        when (ann.desc) {
            "Ldev/memoize/annotations/Memoize;" -> {
                var maxSize = 128
                // Read annotation parameters
                if (ann.values != null) {
                    var i = 0
                    while (i < ann.values.size - 1) {
                        if (ann.values[i] == "maxSize") maxSize = ann.values[i + 1] as? Int ?: 128
                        i += 2
                    }
                }
                memoizedMethods.add(MemoMethodInfo(method.name, method.desc, maxSize))
            }
            "Ldev/memoize/annotations/CacheInvalidate;" -> {
                invalidateMethods.add(method.name)
            }
        }
    }
}
```

The annotation descriptor format (`Ldev/memoize/annotations/Memoize;`) is the JVM internal name. Annotation values are stored as alternating name-value pairs in the ASM tree model.

### 6.5 Phase 2: Field Addition

For a class with `search()` and `length()` memoized:

```kotlin
// Add: private MemoCacheManager __memoCacheManager;
classNode.fields.add(FieldNode(ACC_PRIVATE | ACC_SYNTHETIC, "__memoCacheManager", ...))

// Add: private MemoDispatcher __memoDispatcher_search;
classNode.fields.add(FieldNode(ACC_PRIVATE | ACC_SYNTHETIC, "__memoDispatcher_search", ...))

// Add: private MemoDispatcher __memoDispatcher_length;
classNode.fields.add(FieldNode(ACC_PRIVATE | ACC_SYNTHETIC, "__memoDispatcher_length", ...))
```

Fields are marked `ACC_SYNTHETIC` so they don't appear in IDE auto-complete or Javadoc.

### 6.6 Phase 3: Constructor Patching

Before every `RETURN` instruction in every constructor, the following bytecode is inserted:

```java
// Equivalent Java (ASM generates the bytecode instructions directly):
this.__memoCacheManager = new MemoCacheManager();

this.__memoDispatcher_search = new MemoDispatcher("search", 64);
this.__memoCacheManager.register("search", this.__memoDispatcher_search);

this.__memoDispatcher_length = new MemoDispatcher("length", 128);
this.__memoCacheManager.register("length", this.__memoDispatcher_length);
```

The `maxSize` value (64, 128) comes from the annotation parameter read in Phase 1.

### 6.7 Phase 4: Method Instrumentation

For each `@Memoize` method, the `MemoizeMethodAdapter` (extending ASM's `AdviceAdapter`) injects code at two points:

**At method entry (`onMethodEnter`):**

```java
// Equivalent Java for: @Memoize public boolean search(int key)

// 1. Build cache key from arguments
CacheKeyWrapper __key = this.__memoDispatcher_search.buildKey(
    new Object[]{ Integer.valueOf(key) }    // box primitive args
);

// 2. Check cache
Object __cached = this.__memoDispatcher_search.getIfCached(__key);

// 3. If hit, unwrap and return
if (__cached != null) {
    return ((Boolean) MemoDispatcher.unwrap(__cached)).booleanValue();
}

// 4. Cache miss: fall through to original method body
```

**Before each return (`onMethodExit`):**

```java
// Return value is on the stack. Duplicate it, box, and store in cache.
// (The original return value remains on the stack for the actual return.)
this.__memoDispatcher_search.putInCache(__key, Boolean.valueOf(returnValue));
```

**Primitive handling:** The ASM adapter handles boxing/unboxing for all 8 Java primitive types:

| Return Type | Boxing | Unboxing |
|-------------|--------|----------|
| `boolean` | `Boolean.valueOf(z)` | `((Boolean)obj).booleanValue()` |
| `int` | `Integer.valueOf(i)` | `((Integer)obj).intValue()` |
| `long` | `Long.valueOf(l)` | `((Long)obj).longValue()` |
| `double` | `Double.valueOf(d)` | `((Double)obj).doubleValue()` |
| `Object` / arrays | no-op | `checkcast` to expected type |
| *(all other primitives handled similarly)* | | |

**For `@CacheInvalidate` methods (`InvalidateMethodAdapter`):**

```java
// Injected before each return:
if (this.__memoCacheManager != null) {
    this.__memoCacheManager.invalidateAll();
}
```

The null check handles the edge case where `@CacheInvalidate` is on a class that has no `@Memoize` methods.

## 7. KSP Compile-Time Validation

**Files:**
- [`memoize-ksp/.../MemoizeProcessorProvider.kt`](memoize-ksp/src/main/kotlin/dev/memoize/ksp/MemoizeProcessorProvider.kt)
- [`memoize-ksp/.../MemoizeProcessor.kt`](memoize-ksp/src/main/kotlin/dev/memoize/ksp/MemoizeProcessor.kt)

The KSP processor runs during Kotlin compilation and validates `@Memoize` usage:

| Check | Severity | Condition |
|-------|----------|-----------|
| Void/Unit return | **Error** | `@Memoize` on a method returning `void`/`Unit` |
| Abstract method | **Error** | `@Memoize` on an `abstract` method |
| Non-positive maxSize | **Error** | `maxSize <= 0` in annotation parameters |
| Mutable parameter type | **Warning** | Parameter type not in the immutable set and no `@CacheKey` annotation |

The immutable type set includes: all Kotlin/Java primitives and wrappers, `String`, `BigDecimal`, `BigInteger`, `UUID`, and Kotlin unsigned types.

The KSP processor is **optional** -- the ASM transformation works without it. It provides earlier, more readable error messages than what would happen at runtime.

## 8. End-to-End Walkthrough

Here is the complete lifecycle of a memoized method, from source code to runtime execution:

### Step 1: Developer writes annotated code

```java
// LinkedList.java
public class LinkedList {
    private Node head;

    @Memoize(maxSize = 64)
    public boolean search(int key) {
        Node current = head;
        while (current != null) {
            if (current.data == key) return true;
            current = current.next;
        }
        return false;
    }

    @CacheInvalidate
    public void insert(int data) {
        // ... modifies head ...
    }
}
```

### Step 2: KSP validates at compile time

The KSP processor checks:
- `search` returns `boolean` (not void) -- OK
- `search` is not abstract -- OK
- `maxSize = 64 > 0` -- OK
- Parameter `int key` is in the immutable set -- OK (no warning)

### Step 3: Javac/Kotlinc compiles to `.class` bytecode

Standard compilation produces `LinkedList.class` with `@Memoize` and `@CacheInvalidate` annotations embedded in the bytecode (retention = CLASS).

### Step 4: AGP calls the Memoize Gradle plugin

During the `transformDebugClassesWithAsm` task, the AGP Instrumentation API passes every project `.class` file through `MemoizeClassVisitorFactory`.

### Step 5: MemoizeClassVisitor transforms the bytecode

**Pass 1 (Tree API):**
1. Reads `LinkedList.class` into a `ClassNode`
2. Scans: finds `search` has `@Memoize(maxSize=64)`, `insert` has `@CacheInvalidate`
3. Adds fields: `__memoCacheManager`, `__memoDispatcher_search`
4. Patches constructor: inserts `new MemoCacheManager()`, `new MemoDispatcher("search", 64)`, `register()`

**Pass 2 (Visitor API):**
1. Re-reads the modified bytecode
2. For `search`: injects cache-check at entry, cache-store before returns
3. For `insert`: injects `cacheManager.invalidateAll()` before returns

### Step 6: Transformed bytecode is packaged into the APK

The transformed `.class` files are DEXed and included in the APK. The original source remains unchanged.

### Step 7: Runtime execution

```
list.search(10)          // First call
  → buildKey([10])       // Create CacheKeyWrapper for args
  → getIfCached(key)     // Cache miss (returns null)
  → [original body runs] // Traverses linked list, returns true
  → putInCache(key, true)// Store result
  → return true

list.search(10)          // Second call (same args)
  → buildKey([10])       // Same key
  → getIfCached(key)     // Cache HIT (returns Boolean.TRUE)
  → unwrap(cached)       // Returns the Boolean
  → return true          // Short-circuits, original body never runs

list.insert(20)          // Mutation
  → [original body runs] // Modifies head
  → cacheManager.invalidateAll()  // Clears search cache

list.search(10)          // After mutation
  → getIfCached(key)     // Cache miss (was invalidated)
  → [original body runs] // Re-traverses, returns fresh result
  → putInCache(key, true)
  → return true
```

### Actual transformed bytecode

The `search` method's bytecode after transformation (from `javap -c`):


## 9. Integration Guide

### Adding to an Android project

**1. Settings file** (`settings.gradle.kts`):

```kotlin
pluginManagement {
    includeBuild("path/to/memoize-lib")  // or use mavenLocal/maven repo
    repositories {
        gradlePluginPortal()
        google()
        mavenCentral()
        mavenLocal()
    }
}

dependencyResolutionManagement {
    repositories {
        google()
        mavenCentral()
        mavenLocal()
    }
}
```

**2. App build file** (`app/build.gradle.kts`):

```kotlin
plugins {
    id("com.android.application")
    id("org.jetbrains.kotlin.android")
    id("dev.memoize")                  // Apply the memoize plugin
}

dependencies {
    implementation("dev.memoize:memoize-annotations:0.1.0")
    implementation("dev.memoize:memoize-runtime:0.1.0")
}
```

**3. Annotate methods:**

```java
import dev.memoize.annotations.Memoize;
import dev.memoize.annotations.CacheInvalidate;

public class MyService {
    @Memoize(maxSize = 256)
    public Result computeExpensive(String input) { ... }

    @CacheInvalidate
    public void updateData(String newData) { ... }
}
```

That's it. No call-site changes, no wrapper classes.

## 11. Current Limitations

### Functional limitations

| Limitation | Detail | Workaround |
|------------|--------|------------|
| **Android-only** | The Gradle plugin uses the AGP Instrumentation API, which only works for Android projects. Plain JVM Gradle projects are not supported. | For JVM projects, use the runtime library directly with manual caching. A JVM Gradle Transform plugin could be added. |
| **Instance scope only** | `CacheScope.CLASS` (static caches) is defined in the annotation API but not implemented in the ASM transformation. All caches are currently per-instance. | Use singleton patterns or application-level DI to share instances. |
| **`@CacheKey` field extraction not implemented** | The `@CacheKey("field")` annotation is defined but the ASM transformation does not extract fields. All parameters are used as-is via `Arrays.deepHashCode`. | Ensure parameter types have correct `hashCode()`/`equals()`. Use data classes in Kotlin. |
| **`expireAfterWrite` / `threadSafety` / `eviction` not read from annotation** | The ASM transformation currently only reads `maxSize` from the annotation. Other parameters use defaults. | Extend the annotation parameter reading in `MemoizeClassVisitor` Phase 1. |
| **No `onTrimMemory` integration** | The `MemoCacheManager` does not register for Android memory pressure callbacks. | Can be added by registering with `Application.registerComponentCallbacks()`. |
| **No ProGuard/R8 keep rules shipped** | Minified builds may strip runtime classes or renamed fields. | Add keep rules manually or ship them with the library artifact. |

### Technical limitations

| Limitation | Detail |
|------------|--------|
| **Boxing overhead** | Every memoized call boxes primitive arguments to `Object[]` and the return value to `Object`. For `boolean search(int)`: one `Integer.valueOf()` + one `Boolean.valueOf()` per call. Negligible for expensive computations but measurable for very cheap methods. |
| **No suspend function support** | Kotlin `suspend` functions compile to state machines at bytecode level. The ASM injection does not handle this. |
| **Method overloading** | Dispatcher field names are based on method name only (`__memoDispatcher_search`). Two overloaded methods with the same name but different parameter types would conflict. |
| **Debugging** | Breakpoints in memoized methods will hit the injected cache-check bytecode first. The source code does not match the executed bytecode, which can be confusing in debuggers. |
| **Unit test coverage** | Android unit tests (`testDebugUnitTest`) run against pre-transformation bytecode. True end-to-end testing of the ASM transformation requires either Android instrumented tests (on device/emulator) or a standalone JVM test that manually applies the transformation. |
| **Single RETURN in @CacheInvalidate** | The `InvalidateMethodAdapter` injects before every `RETURN`, which works correctly for multiple return points, but `ATHROW` (exception) paths do NOT trigger invalidation. This is intentional: if a mutating method throws, the state may not have changed. |

### Performance characteristics

| Metric | Value |
|--------|-------|
| Cache hit overhead (single int arg) | ~50-80 ns (key construction + HashMap lookup + unboxing) |
| Cache miss overhead | ~80-120 ns (key construction + miss + store) |
| Memory per dispatcher | ~200 bytes base + entries |
| Memory per cached entry | ~80-120 bytes (key wrapper + boxed value + LinkedHashMap node) |
| Build time impact | < 1 second for typical projects (ASM processes only annotated classes) |
